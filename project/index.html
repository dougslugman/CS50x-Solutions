Perlin Noise Generator & Visualiser




This website allows you to select parameters to put into the Perlin Noise algorithm to generate a map! The Perlin Noise algorithm creates pseudo random terrain that looks realistic. This can be used for a variety of things, such as 3D procedural terrain generation, adding texture to objects and generating 2D maps. There are also different ways to display the values made by the algorithm, including greyscale, black & white and colour. you can also download the generated image! Scroll down to use the website and read more about how to properly use it, if you are unfamiliar.



To download the image, right click the Canvas above and click "Save Image As" and select the place to store it. It is downloaded as a .PNG

Map Display Format

you can select a different display format by clicking on the corresponding buttons below, read more about it and potentially change some factors about the display. Once you have selected the appropriate display time and the facotrs of the display, hit the "Generate Map" to visualise the output in the canvas above!

This display format will use greyscale to represent the value of the pixels. how large the pixels value is (represented as a percentage of its value between the minimum and maximum values being -1 and +1) will directly set the brightness of the pixel. for example, a pixel with a value of -1 would have a percentage value of 0.00 meaning the pixels value would be completely black. a value of -0.5 would have a percentage of 0.25 setting the pixels colour to be 25% bright, or more specifically a hex code of #555555. This is the simplest way to represent the change in value, lighter meaning larger and darker meaning smaller.
This display format will use black and white to display a range of pixel values. all values below a set boundary will be black and all above will be white. By default, this value will be 0, halfway between all possible values however this boundary can be changed with the slider below to any value within -1 and +1

Information & How to Use


This Website won't go into detail for how the algorithm works, so for more information on that read the Wikipedia and watch this Youtube Video to understand how it works and what each parameter means. below will explain the possible ranges of values for all parameters, how each parameter effects the other parameters and how to properly enter it into the inputs above:

Seed value: The seed value will determine the influence vectors for the chunks on the map. The algorithm uses a PRNG meaning the values generated are deterministic, resulting in the seed always giving the same output. You can choose a INTEGER value for the seed between 0 and 4294967295 inclusive (otherwise known as the 32-bit integer limit).

chunk side length & map side length: Both of these parameters are the number of pixels on the chunk/map both width and height wise. This determines the size of the map, the size of chunks and the amount of chunks in the map. NOTE: you must select a chunk length that fits into the map length an integer number of times otherwise the algorithm will reject the input. Also note that (will be explained more further down) the lacunarity and octave count will scale the chunks size so the chunk lengths for ALL octaves must also fit into the map’s length an integer number of times. The number of chunks per map side for the largest octave is calculated as:
ChunksPerSide = ChunkLength x (Lactunarity ^ (OctaveNumber - 1 ))
If chunksPerSide is not an integer for the LARGEST octave, your input will be rejected. both ChunkLength and MapLength must be inputed as INTEGERS larger than 3.

Lacunarity: The lacunarity is how much the chunk length increases by for each Octave. The first octave will not change the chunk length, but the successive octaves will, raised to the power of the current octave number. This influences how quickly chunks grow per octave. The lacunarity must be entered as a non-negative NUMERIC value.

Persistence: The persistence parameter determines how much values in successive octaves effect the overall map. like the lacunarity, it is calculated for each octave by raising it to the power of the current octave (- 1). for example, the 3rd octave with a persistence of 0.5 will be calculated as
0.5 ^ (3 - 1), or 0.5 x 0.5 = 0.25 resulting in all values within the 3rd octave only effecting the map by 25% of its original value. The persistence value is USUALLY one over the chosen lacunarity value (1/Lacunarity). HOWEVER, the lacunarity can be entered as a non-negative NUMERIC value but it is HIGHLY RECOMMENED the value is a FLOAT between 0 and 1. Otherwise, strange maps will be created as the value wasn't designed to exceed 1 but it can if desired.

Octave number: The octave parameters determines how many successive Perlin Noise maps will be generated and overlayed on each other. This value also indirectly effects how the lacunarity and persistence parameters are depending on the octave. The size of maps for successive octaves won’t change, but it will instead affect the sizes of chunks (more specifically, the lacunarity will do it related to the octave number). Octaves can be entered as a non-negative INTEGER that is above or equal to 1.
